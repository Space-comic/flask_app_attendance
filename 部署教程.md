# 基于MySQL和OpenCV的人脸识别考勤系统 — 部署教程

## 系统概述

本系统是一个 BS 架构（浏览器/服务器）的人脸识别考勤管理系统，基于 Flask + MySQL + OpenCV + DeepFace 构建，支持：

- 活体检测（EAR 眨眼算法，防照片/视频攻击）
- 人脸识别签到
- 合照批量识别
- 情绪分析统计
- 教师/学生权限分离
- 考勤记录 Excel 导出

## 二、前置准备

### 2.1 安装 MySQL

1. 下载并安装 [MySQL Community Server](https://dev.mysql.com/downloads/mysql/)
2. 安装时设置 root 密码（默认配置使用 `ltc19981118`，可在 `config.py` 中修改）
3. 确认 MySQL 服务已启动

### 2.2 下载 dlib 人脸特征点模型

活体检测依赖 `shape_predictor_68_face_landmarks.dat` 文件：

1. 访问 http://dlib.net/files/shape_predictor_68_face_landmarks.dat.bz2 下载
2. 解压后将 `.dat` 文件放到 `D:\shape_predictor_68_face_landmarks.dat`
   （或修改 `config.py` 中的 `DLIB_MODEL_PATH` 为实际路径）

## 四、数据库初始化

### 4.1 修改数据库连接配置

打开 `config.py`，根据实际情况修改：

```python
SQLALCHEMY_DATABASE_URI = 'mysql+pymysql://root:你的密码@localhost:3306/face_attendance'
DLIB_MODEL_PATH = r'D:\shape_predictor_68_face_landmarks.dat'  # 修改为实际路径
```

### 4.2 创建数据库

在 MySQL 中执行：

```sql
CREATE DATABASE face_attendance CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 4.3 运行建表脚本

```bash
cd "e:\基于Mysql和OpenCV的人脸识别系统（源码和部署教程）\code\flask_app"
python init_db.py
```

执行成功后会创建以下数据表：
- `users` — 用户表（学生+教师）
- `attendance_records` — 考勤记录
- `emotion_records` — 情绪记录
- `activities` — 活动（合照）
- `activity_participants` — 活动参与者
- `liveness_sessions` — 活体检测会话

### 6.1 注册教师账号

## 七、功能使用说明

### 7.1 添加学生信息（教师操作）

1. 登录教师账号
2. 进入 **管理 → 用户管理**
3. 点击"添加用户"，填写学生学号、姓名等信息
4. 上传学生人脸照片（需保证照片清晰，一人一张）

### 7.2 人脸签到（学生操作）

1. 学生登录账号，进入签到页面
2. 点击"开始活体检测"，对着摄像头**眨眼3次**
3. 活体验证通过后，点击"手动签到"或开启"自动识别"
4. 系统自动识别人脸并记录签到+情绪

### 7.3 合照识别（教师操作）

1. 进入 **合照识别** 页面
2. 输入活动名称，上传合照
3. 系统自动识别照片中所有人脸，生成参与名单

### 7.4 情绪统计

- 访问 **情绪统计** 页面
- 查看饼图（7种情绪分布）和柱状图（时间趋势）
- 数据来源：签到时和合照识别时自动记录

### 7.5 导出考勤记录（教师操作）

在 **考勤记录** 页面，点击"导出 Excel"按钮，下载 `.xlsx` 格式的考勤报表。

## 九、常见问题排查

### Q1: 启动时报 `ModuleNotFoundError: No module named 'dlib'`

**解决：**
```bash
pip install cmake
pip install dlib
```
若仍失败，下载预编译 wheel：https://github.com/jloh02/dlib/releases

### Q2: 启动时报数据库连接错误

**检查：**
1. MySQL 服务是否启动：`net start mysql`
2. `config.py` 中用户名密码是否正确
3. 数据库 `face_attendance` 是否已创建

### Q3: 人脸识别准确率低

**优化建议：**
1. 注册照片保证正脸、光线充足、背景简洁
2. 每人可注册多张不同角度的照片
3. 签到时保持摄像头距离适中（40-80cm）

### Q4: 活体检测无法通过

**检查：**
1. 确认 `shape_predictor_68_face_landmarks.dat` 路径正确
2. 摄像头分辨率需 ≥ 480p
3. 光线充足，避免逆光

### Q5: DeepFace 情绪分析很慢

**说明：** 首次运行 DeepFace 会下载模型文件（约 200MB），后续会缓存，速度正常。

### Q6: 端口 5000 被占用

修改 `app.py` 最后一行：

```python
app.run(debug=False, host='0.0.0.0', port=8080)  # 改为其他端口
```
